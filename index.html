Real-Time Git Command Highlighting

git-highlight is a Zsh plugin that visually guides you through your Git commands as you type. It highlights valid and partially matched Git subcommands, and flags invalid ones right away.

Perfect for those late-night sessions when you're not at your sharpest. With git-highlight, you'll catch typos before running the wrong command.

Example:

Typing: git sta will highlight git in green and sta in yellow, hinting that you meant status.

Once you complete it to: git status, both words will be in green, confirming a valid command.

Installation

  1. Clone the repo: 
    git clone https://github.com/patriciosebastian/git-highlight.git ~/.git-highlight
  2. Source the script in your .zshrc: 
    echo "source ~/.git-highlight/git-highlight.zsh" >> ~/.zshrc
source ~/.zshrc

From now on, every new terminal session will have real-time Git command highlighting enabled.

Using a Zsh Plugin Manager

If you're using a Zsh plugin manager like zplug, just add: 

zplug "patriciosebastian/git-highlight"

Customization

Inside git-highlight.zsh, you'll find:

**Git Commands:** Modify the validGitCommands array to add or remove recognized commands: 

typeset -a validGitCommands=(git init commit push pull status checkout merge branch fetch rebase diff add log stash pop restore reset)

Add more: 

validGitCommands+=(blame cherry-pick worktree)

**Shell Operators:** If you want to handle more or fewer operators, update the case block: 

case "$word" in
  '&&'|'||'|';'|'&')
    expect_subcommand=false
    ;;
esac

Feel free to tweak these as needed for your workflow.
